Python

Python is a high-level, interpreted programming language known for its readability and versatility. It is widely used in web development, data science, artificial intelligence, and more.

Syntax

Python syntax is designed to be readable and straightforward. It uses indentation to define code blocks.

def greet(name):
            print(f"Hello, {name}!")
        
        greet("World")

Examples

Here is a simple Python example that prints "Hello, World!" to the console:

print("Hello, World!")

Data Types

Python supports various data types including:

  • Numbers: Integers, floating-point numbers, and complex numbers.
  • Strings: Immutable sequences of characters.
  • Lists: Mutable sequences of elements.
  • Tuples: Immutable sequences of elements.
  • Dictionaries: Unordered collections of key-value pairs.
  • Sets: Unordered collections of unique elements.
  • Booleans: True or False values.
  • None: Represents absence of a value.

Functions

Functions in Python allow code reuse and modularization. They can take arguments and return values:

def greet(name):
            print(f"Hello, {name}!")
        
        greet("World")

Python also supports anonymous functions using the lambda keyword:

add = lambda x, y: x + y
        print(add(3, 5))  # Output: 8

Control Flow

Python supports various control flow statements including:

  • if...elif...else: Conditional statements for branching.
  • for: Iterating over sequences.
  • while: Looping based on a condition.
  • break: Exiting a loop prematurely.
  • continue: Skipping the rest of the current iteration.
  • pass: Placeholder indicating no action.

Modules and Packages

Python allows organizing code into modules and packages for better organization and reuse. Modules are files containing Python code, while packages are directories containing modules and an __init__.py file:

import math
        print(math.sqrt(25))  # Output: 5.0

Exception Handling

Python provides try...except blocks for handling exceptions and errors:

try:
            x = 1 / 0
        except ZeroDivisionError:
            print("Cannot divide by zero!")

File Handling

Python allows reading from and writing to files:

with open("example.txt", "r") as file:
            print(file.read())

Classes and Objects

Python supports object-oriented programming with classes and objects:

class Person:
            def __init__(self, name, age):
                self.name = name
                self.age = age
        
            def greet(self):
                print(f"Hello, my name is {self.name} and I am {self.age} years old.")
        
        person = Person("Alice", 30)
        person.greet()

Regular Expressions

Python provides the re module for working with regular expressions:

import re
        pattern = r"\b\w{3}\b"
        text = "Python is a programming language"
        matches = re.findall(pattern, text)
        print(matches)  # Output: ['Python', 'is', 'programming', 'language']

Additional Documentation

For more detailed documentation, refer to the official Python documentation.

JavaScript

JavaScript is a versatile language commonly used in web development for both client-side and server-side programming. It is essential for creating interactive web applications.

Syntax

JavaScript syntax is dynamic and can be used in both functional and object-oriented programming styles.

function greet(name) {
            console.log(`Hello, ${name}!`);
        }
        
        greet("World");

Examples

Here is a simple JavaScript example that prints "Hello, World!" to the console:

console.log("Hello, World!");

Data Types

JavaScript supports various data types including:

  • Numbers: Integers, floating-point numbers, and special numeric values like Infinity and NaN.
  • Strings: Immutable sequences of characters.
  • Booleans: True or False values.
  • Objects: Collections of key-value pairs.
  • Arrays: Ordered collections of elements.
  • Undefined: Indicates a variable that has not been assigned a value.
  • null: Represents the absence of any object value.

Functions

Functions in JavaScript allow code reuse and modularization. They can take arguments and return values:

function add(x, y) {
            return x + y;
        }
        
        console.log(add(3, 5));  // Output: 8

JavaScript also supports anonymous functions using the function keyword:

const greet = function(name) {
            console.log(`Hello, ${name}!`);
        }
        
        greet("World");

Control Flow

JavaScript supports various control flow statements including:

  • if...else: Conditional statements for branching.
  • for: Iterating over sequences.
  • while: Looping based on a condition.
  • break: Exiting a loop prematurely.
  • continue: Skipping the rest of the current iteration.
  • switch: Multi-way branch statement.

Objects and Classes

JavaScript supports object-oriented programming with objects and classes:

class Person {
            constructor(name, age) {
                this.name = name;
                this.age = age;
            }
        
            greet() {
                console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
            }
        }
        
        const person = new Person("Alice", 30);
        person.greet();

Modules and Packages

JavaScript allows organizing code into modules and packages for better organization and reuse:

// Module: math.js
        export function add(x, y) {
            return x + y;
        }
        
        // Module: main.js
        import { add } from './math.js';
        console.log(add(3, 5));  // Output: 8

Error Handling

JavaScript provides try...catch blocks for handling exceptions and errors:

try {
            const result = 1 / 0;
        } catch (error) {
            console.log("Error:", error.message);
        }

Asynchronous Programming

JavaScript supports asynchronous programming using callbacks, promises, and async/await:

// Callback example
        function fetchData(callback) {
            setTimeout(() => {
                callback("Data fetched");
            }, 1000);
        }
        
        fetchData((data) => {
            console.log(data);
        });
        
        // Promise example
        function fetchData() {
            return new Promise((resolve, reject) => {
                setTimeout(() => {
                    resolve("Data fetched");
                }, 1000);
            });
        }
        
        fetchData().then((data) => {
            console.log(data);
        });
        
        // Async/Await example
        async function fetchData() {
            return new Promise((resolve, reject) => {
                setTimeout(() => {
                    resolve("Data fetched");
                }, 1000);
            });
        }
        
        async function fetchDataAndLog() {
            const data = await fetchData();
            console.log(data);
        }
        
        fetchDataAndLog();

DOM Manipulation

JavaScript is used to manipulate the Document Object Model (DOM) of web pages, allowing dynamic interaction with HTML elements:

// Change text content
        document.getElementById("demo").textContent = "Hello, World!";
        
        // Add event listener
        document.getElementById("btn").addEventListener("click", () => {
            alert("Button clicked!");
        });

Web APIs

JavaScript interacts with various Web APIs to access browser functionalities:

// Fetch API for making HTTP requests
        fetch('https://api.example.com/data')
            .then(response => response.json())
            .then(data => console.log(data));
        
        // Local Storage API for storing data locally
        localStorage.setItem('key', 'value');
        const storedValue = localStorage.getItem('key');

Asynchronous JavaScript

Asynchronous JavaScript allows performing tasks without blocking the execution of other operations:

// setTimeout
        console.log("Start");
        setTimeout(() => {
            console.log("Inside setTimeout");
        }, 2000);
        console.log("End");
        
        // setInterval
        setInterval(() => {
            console.log("Interval callback");
        }, 1000);

Additional Resources

For more detailed documentation and tutorials, refer to the MDN Web Docs for JavaScript.

HTML (HyperText Markup Language)

HTML is the standard markup language for creating web pages. It describes the structure of web pages using markup.

Syntax

HTML syntax consists of elements represented by tags.

<!DOCTYPE html>
        <html>
        <head>
            <title>Title of the document</title>
        </head>
        <body>
            <h1>This is a heading</h1>
            <p>This is a paragraph.</p>
        </body>
        </html>

Elements

HTML documents are composed of various elements:

  • Headings: <h1> to <h6> for defining headings.
  • Paragraphs: <p> for defining paragraphs.
  • Links: <a> for creating hyperlinks.
  • Images: <img> for embedding images.
  • Lists: <ul> and <ol> for unordered and ordered lists.
  • Forms: <form>, <input>, <select>, etc., for user input.
  • Tables: <table>, <tr>, <td>, etc., for tabular data.
  • Divisions: <div> for grouping elements.
  • Attributes: HTML elements can have attributes like id, class, src, etc.

Attributes

HTML elements can have various attributes to provide additional information or behavior:

  • id: Specifies a unique id for an element.
  • class: Specifies one or more class names for an element.
  • src: Specifies the URL of an external resource like an image or script.
  • href: Specifies the URL of a hyperlink.
  • alt: Specifies an alternate text for an image.
  • title: Specifies extra information about an element (tooltip).
  • type: Specifies the type of an input element.
  • value: Specifies the value of an input element.
  • placeholder: Specifies a placeholder text for an input element.

Forms

HTML forms are used to collect user input. They consist of various form elements like text fields, checkboxes, radio buttons, etc.

<form action="/submit" method="post">
            <label for="username">Username:</label>
            <input type="text" id="username" name="username">
            <input type="submit" value="Submit">
        </form>

Tables

HTML tables are used to display tabular data:

<table>
            <tr>
                <th>Name</th>
                <th>Age</th>
            </tr>
            <tr>
                <td>John</td>
                <td>30</td>
            </tr>
            <tr>
                <td>Jane</td>
                <td>25</td>
            </tr>
        </table>

Metadata

HTML documents can contain metadata, such as character encoding, viewport settings, etc.:

<!DOCTYPE html>
        <html lang="en">
        <head>
            <meta charset="UTF-8">
            <meta name="viewport" content="width=device-width, initial-scale=1.0">
            <title>Title of the document</title>
        </head>
        <body>
            ...
        </body>
        </html>

Links and Navigation

HTML links are used for navigation:

<a href="/page1">Page 1</a>

Audio and Video

HTML supports embedding audio and video content:

<audio controls>
            <source src="audio.mp3" type="audio/mpeg">
            Your browser does not support the audio element.
        </audio>
        
        <video controls>
            <source src="video.mp4" type="video/mp4">
            Your browser does not support the video element.
        </video>

Canvas

HTML5 introduced the <canvas> element for drawing graphics using JavaScript:

<canvas id="myCanvas" width="200" height="100">
            Your browser does not support the canvas element.
        </canvas>

CSS (Cascading Style Sheets)

CSS is a stylesheet language used to describe the presentation of a document written in HTML or XML. CSS describes how elements should be rendered on screen, on paper, or in other media.

Syntax

CSS syntax consists of selectors and declarations. A declaration block contains one or more declarations separated by semicolons.

h1 {
            color: blue;
            font-size: 24px;
        }

Selectors

Selectors are used to target HTML elements for styling:

  • Element Selector: Targets all instances of a specified HTML element.
  • Class Selector: Targets elements with a specific class attribute.
  • ID Selector: Targets a single element with a specific ID attribute.
  • Attribute Selector: Targets elements based on their attributes.
  • Descendant Selector: Targets elements that are descendants of a specified element.
  • Child Selector: Targets elements that are direct children of a specified element.
  • Adjacent Sibling Selector: Targets an element that is directly preceded by a specified element.
  • Pseudo-classes and Pseudo-elements: Adds special effects to selected elements.

Properties and Values

CSS properties define how HTML elements are displayed:

  • Color: Sets the color of text.
  • Font-family: Specifies the font family for text.
  • Font-size: Sets the size of text.
  • Background-color: Sets the background color of an element.
  • Width and Height: Sets the width and height of an element.
  • Margin and Padding: Adds space around or inside an element.
  • Border: Sets the border of an element.
  • Position: Specifies the positioning method of an element.
  • Display: Defines the display behavior of an element.
  • Float: Positions an element to the left or right within its container.

CSS Box Model

The CSS box model describes the design and layout of elements in CSS:

  • Content: The content of the box, where text and images appear.
  • Padding: Transparent area around the content, inside the border.
  • Border: The border of the box.
  • Margin: Transparent area outside the border, between the border and adjacent elements.

Positioning

CSS offers different positioning schemes for elements:

  • Static: Default positioning; elements are positioned according to the normal flow of the document.
  • Relative: Positions an element relative to its normal position.
  • Absolute: Positions an element relative to its nearest positioned ancestor.
  • Fixed: Positions an element relative to the browser window.

Responsive Design

CSS can be used to create responsive designs that adapt to different screen sizes:

  • Media Queries: Conditionally apply CSS based on device characteristics like screen width.
  • Viewport Meta Tag: Sets the viewport width and initial scale.
  • Flexible Layouts: Use percentage widths and flexible units like em and rem.

Animation and Transitions

CSS allows for creating animations and transitions to enhance user experience:

  • Transitions: Smoothly animate changes in CSS properties.
  • Keyframe Animations: Define complex animations using keyframes.

Preprocessors

CSS preprocessors extend the functionality of CSS with features like variables, mixins, and nesting:

  • Sass (Syntactically Awesome Stylesheets): Adds features like variables, nesting, and mixins.
  • Less: Similar to Sass, with features like variables and mixins.
  • Stylus: A more expressive and flexible CSS preprocessor.

Frameworks and Libraries

Frameworks and libraries provide pre-written CSS styles and components for building websites:

  • Bootstrap: A popular CSS framework for building responsive and mobile-first websites.
  • Foundation: Another responsive front-end framework for building websites and web applications.
  • Tailwind CSS: A utility-first CSS framework for rapidly building custom designs.

CSS Grid and Flexbox

CSS Grid and Flexbox are layout models that simplify the design and alignment of elements:

  • CSS Grid: A two-dimensional layout system for creating grid-based designs.
  • Flexbox: A one-dimensional layout model for arranging elements in a row or column.

Vendor Prefixes

Vendor prefixes are used to add browser-specific CSS properties:

  • -webkit-: Safari and Chrome.
  • -moz-: Firefox.
  • -ms-: Internet Explorer.
  • -o-: Opera.

CSS Variables

CSS variables allow for defining reusable values in CSS:

:root {
            --main-color: #336699;
        }
        
        .element

SQL (Structured Query Language)

SQL is a standard language for accessing and manipulating databases. It is used to perform tasks such as retrieving data from a database, inserting data into a database, updating data in a database, and deleting data from a database.

Basic Syntax

SQL syntax typically consists of clauses such as SELECT, FROM, WHERE, GROUP BY, HAVING, ORDER BY, etc.:

SELECT column1, column2
        FROM table_name
        WHERE condition;

Queries

SQL queries are used to retrieve data from a database:

SELECT * FROM table_name;

Joins

SQL JOIN clauses are used to combine rows from two or more tables based on a related column between them:

SELECT Orders.OrderID, Customers.CustomerName
        FROM Orders
        INNER JOIN Customers ON Orders.CustomerID = Customers.CustomerID;

Aggregate Functions

SQL aggregate functions perform calculations on a set of values and return a single value:

  • AVG: Calculates the average value.
  • COUNT: Counts the number of rows.
  • MAX: Returns the maximum value.
  • MIN: Returns the minimum value.
  • SUM: Calculates the sum of values.

Subqueries

SQL subqueries are queries nested within another query:

SELECT column_name(s)
        FROM table_name
        WHERE column_name IN (SELECT column_name
                              FROM table_name
                              WHERE condition);

Constraints

SQL constraints are used to specify rules for data in a table:

  • NOT NULL: Ensures a column cannot have a NULL value.
  • UNIQUE: Ensures that each row in a column is unique.
  • PRIMARY KEY: Uniquely identifies each row in a table.
  • FOREIGN KEY: Enforces a link between data in two tables.
  • CHECK: Ensures that all values in a column satisfy certain conditions.
  • DEFAULT: Specifies a default value for a column when no value is provided.

Transactions

SQL transactions are units of work that are performed as a single logical operation:

BEGIN TRANSACTION;
        -- SQL statements here
        COMMIT;

Indexes

SQL indexes are used to speed up the retrieval of rows from a table:

CREATE INDEX index_name
        ON table_name (column_name);

Views

SQL views are virtual tables based on the result set of a SELECT query:

CREATE VIEW view_name AS
        SELECT column1, column2
        FROM table_name
        WHERE condition;

Stored Procedures

SQL stored procedures are prepared SQL code that can be saved and reused:

CREATE PROCEDURE procedure_name
        AS
        SQL_statements;

Triggers

SQL triggers are special stored procedures that are automatically executed when certain events occur:

CREATE TRIGGER trigger_name
        ON table_name
        FOR INSERT
        AS
        SQL_statements;

Normalization

SQL normalization is the process of organizing data in a database to reduce redundancy and improve data integrity:

  • First Normal Form (1NF): Eliminates repeating groups by putting each value in a separate column.
  • Second Normal Form (2NF): Meets the requirements of 1NF and ensures that non-key attributes are fully functional dependent on the primary key.
  • Third Normal Form (3NF): Meets the requirements of 2NF and removes transitive dependencies.

Views vs. Materialized Views

Materialized views store the result of a query physically, whereas views do not. Materialized views are useful when you need to access precomputed results frequently:

CREATE MATERIALIZED VIEW mv_name AS
        SELECT column1, column2
        FROM table_name
        WHERE condition;

Full-Text Search

Full-text search allows users to search for documents or specific words or phrases:

SELECT *
        FROM table_name
        WHERE MATCH(column_name) AGAINST('search_term');

Window Functions

Window functions perform a calculation across a set of table rows that are related to the current row:

SELECT column1, column2, SUM(column3) OVER (PARTITION BY column1) AS column3_sum
        FROM table_name;

Recursive Queries

Recursive queries are used to query hierarchical data:

WITH RECURSIVE cte AS (
          SELECT id, parent_id
          FROM table_name
          WHERE id = 1
          UNION ALL
          SELECT t.id, t.parent_id
          FROM table_name t
          JOIN cte ON cte.id = t.parent_id
        )
        SELECT *
        FROM cte;
